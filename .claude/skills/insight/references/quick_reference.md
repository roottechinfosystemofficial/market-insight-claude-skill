# Insight 技能 - 快速速查表

## 一句话说明

用AI在10分钟内完成传统方法需要数周的用户洞察分析，直接输出可用于产品设计和营销文案的具体洞察。

---

## 三段式框架速览

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│  阶段一：用户画像  │ -> │  阶段二：情绪洞察  │ -> │ 阶段三：产品机会  │
│                 │    │                 │    │                 │
│ 4类潜在用户      │    │ 痛点/痒点/爽点   │    │ P0/P1/P2优先级  │
│ +核心用户判断    │    │ +可直接用的文案   │    │ +可落地的清单    │
└─────────────────┘    └─────────────────┘    └─────────────────┘
```

## 核心概念对比

| 传统方法 | AI方法 |
|---------|--------|

## 三个关键维度

### 痛点 Pain Point
用户最担心/恐惧的是什么？
- 关键：找到"被背叛"的情绪触发点
- 竞争对手往往不是同类产品，而是记忆/习惯

### 痒点 Itch Point
用户想表达什么样的身份？
- 关键：理解"理想中的自己"这种心理投射
- 用户希望别人如何看待他的选择

### 爽点 Pleasure Point
那个"啊~"的满足瞬间是什么？
- 关键：具体的生理/心理细节
- 温度、口感、节奏、情感释放

## 快速启动流程

```
用户输入产品/服务
        ↓
【阶段一】生成4类用户画像 + 核心用户推荐
        ↓
用户选择要分析的用户群
        ↓
【阶段二】输出痛点/痒点/爽点 + 可用文案
        ↓
【阶段三】输出P0/P1/P2产品机会清单
        ↓
得到：明天就能验证的具体方向
```

---

## 经典案例：胡辣汤

| 阶段 | 核心输出 |
|------|----------|
| 用户画像 | "碳水充能打工人"为核心（一周3-5次，高频复购） |
| 痛点 | "温吞的改良是对记忆的双重背叛" |
| 痒点 | "粗粝的辛辣证明没被生活磨平" |
| 爽点 | "胡椒逼汗，灵魂归位" |
| P0机会 | "原教旨主义胡辣汤" - 拒绝改良作为卖点 |
| P1机会 | "冰火回魂套餐" - 85℃汤 + 0-4℃沙 |

## 输出质量检查清单

- [ ] 用户描述有画面感（能看到具体的人）
- [ ] 核心用户选择有商业逻辑支撑
- [ ] 痛点/痒点/爽点有情绪共鸣
- [ ] 文案表达可直接用于营销
- [ ] 产品机会具体可执行
- [ ] 优先级排序有理由说明

## 常见问题

**Q: AI生成的洞察能直接用吗？**
A: 不是"最终答案"，而是"快速可验证的方向"。需要结合真实业务场景测试迭代。

**Q: 适合什么行业？**
A: 通用框架，可适配餐饮、SaaS、电商、教育等行业。

**Q: 需要什么输入？**
A: 产品/服务描述，以及希望分析的侧重点（可选）。

**Q: 输出格式是什么？**
A: 结构化Markdown报告，可导出为文档或演示文稿。
